# Farm360 AI Copilot: Operations Runbook

## How to Debug Routing Failures
If a query like "What is the price of cotton?" is failing or returning an `UNKNOWN` intent:
1. **Check `app/router/intents.py`**: Verify that the semantic concepts for "price" and "market" exist within the `MARKET_QUERY` templates.
2. **Review Metrics**: Query Prometheus for `farm360_intent_UNKNOWN_total` to see if there is a sudden spike in unrecognized intents.
3. **Inspect the Tracer Log**: Search the structured logs using the `request_id` to see the exact intent confidence score generated by the router.

## How to Debug Memory Issues
If a follow-up query like "What about Pune?" fails to inherit the `crop` context:
1. **Validate `app/memory/manager.py`**: Ensure the `ContextualMemoryManager` is correctly extracting the missing entities.
2. **Check Session Expiry**: Verify that the `session_id` was identical in both requests and occurred within the 30-minute window.
3. **Database Lookups**: If memory relies on database fetching (e.g., getting a farmer's default location), check `app/repositories/farmer_repo.py` for connection timeouts.

## How to Debug Reasoning Failures
If a complex workflow (e.g., scheme eligibility) hangs or generates an invalid plan:
1. **DAG Validator**: The `PlanValidator` in `app/reasoning/validator.py` enforces acyclic dependencies. Check if the newly added plan introduces a circular loop.
2. **Step Limits**: Ensure the plan does not exceed the `MAX_STEPS` limit (default: 5).
3. **Tool Dependencies**: Check if Step 2 relies on an output variable from Step 1 that was never returned by the execution engine.

## How to Add New Tools
1. **Define the Enum**: Add the new tool to `ToolName` in `app/access/models.py`.
2. **Set Permissions**: Map the required RBAC privileges in `app/access/permissions.py`.
3. **Implement the Logic**: Add the Python function execution logic inside `app/tools/implementations.py`.
4. **Register the Tool**: Map the `ToolName` to the function in the `ToolRegistry` (`app/tools/registry.py`).

## How to Add New Intents
1. **Create Enum**: Add the intent to `Intent` in `app/router/intents.py`.
2. **Update Mapping**: In `app/router/resolver.py`, map the new Intent to the appropriate `ToolName`.
3. **Update LLM Prompts**: Add instruction templates to the generative prompt in `app/response/generator.py` so the model knows how to format the new tool's raw JSON data.

## How to Add New Languages
1. **Update `LanguageEnum`**: Add the locale code (e.g., `GU` for Gujarati) in `app/context/models.py`.
2. **Update System Prompt**: Instruct the LLM in `app/response/generator.py` to synthesize the final output based on the `LanguageEnum` passed in the `UserContext`.
3. *(Optional)* Add few-shot examples for the new language to the Router's embedding dataset if the intent classifier struggles with translation.

## How to Investigate PII Alerts
If an alert is triggered for `PII_LEAK_ATTEMPT`:
1. **Isolate Request**: Find the `request_id` from the AlertManager logs.
2. **Identify Source**: Determine if the PII was inputted by the user (a prompt injection attack) or returned by a backend Tool (an API schema leakage).
3. **Patch**: If it was a backend leak, update the specific tool in `app/tools/implementations.py` to mask the PII before returning the dictionary to the execution engine.
